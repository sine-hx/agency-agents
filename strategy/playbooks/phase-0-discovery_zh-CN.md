# 🔍 阶段 0 操作指南 — 情报收集与发现

> **持续时间**: 3-7 天 | **智能体数量**: 6 | **把关人**: 执行摘要生成器

---

## 目标

在投入资源之前验证机会。在理解问题、市场和监管环境之前，不进行构建。

## 前置条件

- [ ] 项目简报或初始概念已存在
- [ ] 利益相关方赞助人已确定
- [ ] 发现阶段预算已获批

## 智能体激活序列

### 第一波: 并行启动 (第 1 天)

#### 🔍 趋势研究员 — 市场情报主导

```
激活趋势研究员，对 [项目领域] 进行市场情报分析。

所需交付物:
1. 竞争格局分析 (直接竞争对手 + 间接竞争对手)
2. 市场规模估算: TAM、SAM、SOM 及方法论
3. 趋势生命周期映射: 该市场处于采用曲线的哪个位置?
4. 3-6 个月趋势预测，附带置信区间
5. 该领域的投资和融资趋势

来源: 至少 15 个独立、已验证的来源
格式: 附带执行摘要的战略报告
时间线: 3 天
```

#### 💬 反馈综合器 — 用户需求分析

```
激活反馈综合器，对 [项目领域] 进行用户需求分析。

所需交付物:
1. 多渠道反馈收集计划 (问卷、访谈、评论、社交媒体)
2. 现有用户触点的情感分析
3. 痛点识别与优先级排序 (RICE 评分)
4. 功能需求分析，附带商业价值估算
5. 从反馈模式中识别流失风险指标

格式: 附带优先级矩阵的综合反馈报告
时间线: 3 天
```

#### 🔍 UX 研究员 — 用户行为分析

```
激活 UX 研究员，对 [项目领域] 进行用户行为分析。

所需交付物:
1. 用户访谈计划 (5-10 位目标用户)
2. 人物画像开发 (3-5 个主要人物画像)
3. 主要用户流程的旅程映射
4. 竞品产品的可用性启发式评估
5. 附带统计验证的行为洞察

格式: 附带人物画像和旅程地图的研究发现报告
时间线: 5 天
```

### 第二波: 并行启动 (第 1 天，与第一波独立)

#### 📊 分析报告员 — 数据环境评估

```
激活分析报告员，对 [项目领域] 进行数据环境评估。

所需交付物:
1. 现有数据源审计 (有哪些可用数据?)
2. 信号识别 (我们可以测量什么?)
3. 基线指标建立
4. 数据质量评估，附带完整性评分
5. 分析基础设施建议

格式: 附带信号图的数据审计报告
时间线: 2 天
```

#### ⚖️ 法律合规检查器 — 监管扫描

```
激活法律合规检查器，对 [项目领域] 进行监管扫描。

所需交付物:
1. 适用监管框架 (GDPR、CCPA、HIPAA 等)
2. 数据处理要求和约束
3. 目标市场的司法管辖区域映射
4. 合规风险评估，附带严重性评级
5. 阻塞性 vs 可管理的合规问题

格式: 合规要求矩阵
时间线: 3 天
```

#### 🛠️ 工具评估器 — 技术格局评估

```
激活工具评估器，对 [项目领域] 进行技术格局评估。

所需交付物:
1. 问题领域的技术栈评估
2. 关键组件的构建 vs 采购分析
3. 与现有系统的集成可行性
4. 开源 vs 商业方案评估
5. 技术风险评估

格式: 附带建议矩阵的技术栈评估报告
时间线: 2 天
```

## 汇合点 (第 5-7 天)

所有六个智能体提交各自报告。执行摘要生成器进行综合:

```
激活执行摘要生成器，综合阶段 0 的发现。

输入文档:
1. 趋势研究员 → 市场分析报告
2. 反馈综合器 → 综合反馈报告
3. UX 研究员 → 研究发现报告
4. 分析报告员 → 数据审计报告
5. 法律合规检查器 → 合规要求矩阵
6. 工具评估器 → 技术栈评估报告

输出: 执行摘要 (≤500 字，SCQA 格式)
所需决策: GO / NO-GO / PIVOT
包含: 量化的市场机会、已验证的用户需求、监管路径、技术可行性
```

## 质量门检查清单

| # | 标准 | 证据来源 | 状态 |
|---|------|---------|------|
| 1 | 市场机会已验证，TAM > 最低可行阈值 | 趋势研究员报告 | ☐ |
| 2 | ≥3 个已验证用户痛点，附带支持数据 | 反馈综合器 + UX 研究员 | ☐ |
| 3 | 未发现阻塞性合规问题 | 法律合规检查器矩阵 | ☐ |
| 4 | 关键指标和数据源已识别 | 分析报告员审计 | ☐ |
| 5 | 技术栈可行并已评估 | 工具评估器评估 | ☐ |
| 6 | 执行摘要已交付，附带 GO/NO-GO 建议 | 执行摘要生成器 | ☐ |

## 门决策

- **GO**: 进入阶段 1 — 战略与架构
- **NO-GO**: 归档发现，记录经验教训，重新调配资源
- **PIVOT**: 根据发现修改范围/方向，重新运行针对性发现
