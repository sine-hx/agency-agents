# ⚙️ 阶段 2 操作指南 — 基础搭建与脚手架

> **持续时间**: 3-5 天 | **智能体数量**: 6 | **把关人**: DevOps 自动化器 + 证据收集器

---

## 目标

搭建所有后续工作依赖的技术和运营基础。在添加"肌肉"之前先让"骨架"立起来。此阶段完成后，每位开发者都有可运行的环境、可部署的管道和可使用的设计系统。

## 前置条件

- [ ] 阶段 1 轨迹门已通过 (架构包已批准)
- [ ] 阶段 1 移交包已接收
- [ ] 所有架构文档已最终确定

## 智能体激活序列

### 工作流 A: 基础设施 (第 1-3 天，并行)

#### 🚀 DevOps 自动化器 — CI/CD 管道 + 基础设施

```
激活 DevOps 自动化器，为 [项目] 设置基础设施。

输入: 后端架构师系统架构 + 部署要求
所需交付物:
1. CI/CD 管道 (GitHub Actions / GitLab CI)
   - 安全扫描阶段
   - 自动化测试阶段
   - 构建和容器化阶段
   - 部署阶段 (蓝绿部署或金丝雀部署)
   - 自动回滚能力
2. 基础设施即代码 (IaC)
   - 环境配置 (开发、预发布、生产)
   - 容器编排设置
   - 网络和安全配置
3. 环境配置
   - 秘密管理
   - 环境变量管理
   - 多环境一致性

需创建的文件:
- .github/workflows/ci-cd.yml (或等效文件)
- infrastructure/ (Terraform/CDK 模板)
- docker-compose.yml
- Dockerfile(s)

格式: 可运行的 CI/CD 管道，附带 IaC 模板
时间线: 3 天
```

#### 🏗️ 基础设施维护者 — 云基础设施 + 监控

```
激活基础设施维护者，为 [项目] 设置监控。

输入: DevOps 自动化器基础设施 + 后端架构师架构
所需交付物:
1. 云资源配置
   - 计算、存储、网络资源
   - 自动扩展配置
   - 负载均衡器设置
2. 监控栈
   - 应用指标 (Prometheus/DataDog)
   - 基础设施指标
   - 自定义仪表板 (Grafana)
3. 日志和告警
   - 集中式日志聚合
   - 关键阈值的告警规则
   - 值班通知设置
4. 安全加固
   - 防火墙规则
   - SSL/TLS 配置
   - 访问控制策略

格式: 附带仪表板访问权限的基础设施就绪报告
时间线: 3 天
```

#### ⚙️ 运营工作室 — 流程设置

```
激活运营工作室，为 [项目] 设置流程。

输入: 冲刺优先排序器计划 + 项目协调者协调需求
所需交付物:
1. Git 工作流
   - 分支策略 (GitFlow / 主干开发)
   - PR 审查流程
   - 合并策略
2. 沟通渠道
   - 团队频道设置
   - 通知路由
   - 状态更新节奏
3. 文档模板
   - PR 模板
   - Issue 模板
   - 决策日志模板
4. 协作工具
   - 项目看板设置
   - 冲刺跟踪配置

格式: 运营操作手册
时间线: 2 天
```

### 工作流 B: 应用基础 (第 1-4 天，并行)

#### 🎨 前端开发者 — 项目脚手架 + 组件库

```
激活前端开发者，为 [项目] 创建项目脚手架。

输入: UX 架构师 CSS 设计系统 + 品牌守护者身份
所需交付物:
1. 项目脚手架
   - 框架设置 (按架构选择 React/Vue/Angular)
   - TypeScript 配置
   - 构建工具 (Vite/Webpack/Next.js)
   - 测试框架 (Jest/Vitest + Testing Library)
2. 设计系统实施
   - UX 架构师的 CSS 设计令牌
   - 基础组件库 (Button、Input、Card、Layout)
   - 主题系统 (浅色/深色/系统跟随切换)
   - 响应式工具
3. 应用外壳
   - 路由设置
   - 布局组件 (Header、Footer、Sidebar)
   - 错误边界实现
   - 加载状态

需创建的文件:
- src/ (应用源码)
- src/components/ (组件库)
- src/styles/ (设计令牌)
- src/layouts/ (布局组件)

格式: 可运行的应用骨架，附带组件库
时间线: 3 天
```

#### 🏗️ 后端架构师 — 数据库 + API 基础

```
激活后端架构师，为 [项目] 搭建 API 基础。

输入: 系统架构规范 + 数据库 Schema 设计
所需交付物:
1. 数据库设置
   - Schema 部署 (迁移)
   - 索引创建
   - 开发环境的种子数据
   - 连接池配置
2. API 脚手架
   - 框架设置 (Express/FastAPI 等)
   - 与架构匹配的路由结构
   - 中间件栈 (认证、验证、错误处理、CORS)
   - 健康检查端点
3. 认证系统
   - 认证提供商集成
   - JWT/会话管理
   - 基于角色的访问控制脚手架
4. 服务通信
   - API 版本管理设置
   - 请求/响应序列化
   - 错误响应标准化

需创建的文件:
- api/ 或 server/ (后端源码)
- migrations/ (数据库迁移)
- docs/api-spec.yaml (OpenAPI 规范)

格式: 可运行的 API 脚手架，附带数据库和认证
时间线: 4 天
```

#### 🏛️ UX 架构师 — CSS 系统实施

```
激活 UX 架构师，为 [项目] 实施 CSS 系统。

输入: 品牌守护者身份 + 自己阶段 1 的 CSS 设计系统规范
所需交付物:
1. 设计令牌实施
   - CSS 自定义属性 (颜色、字体、间距)
   - 品牌色板，附带语义命名
   - 字体尺度，附带响应式调整
2. 布局系统
   - 容器系统 (响应式断点)
   - Grid 模式 (两列、三列、侧边栏)
   - Flexbox 工具
3. 主题系统
   - 浅色主题变量
   - 深色主题变量
   - 系统偏好检测
   - 主题切换组件
   - 主题间平滑过渡

需创建/更新的文件:
- css/design-system.css (或框架中的等效文件)
- css/layout.css
- css/components.css
- js/theme-manager.js

格式: 已实施的 CSS 设计系统，附带主题切换
时间线: 2 天
```

## 验证检查点 (第 4-5 天)

### 证据收集器验证

```
激活证据收集器，验证阶段 2 基础。

通过截图证据验证以下内容:
1. CI/CD 管道成功执行 (展示管道日志)
2. 应用骨架在浏览器中加载 (桌面截图)
3. 应用骨架在移动设备上加载 (移动端截图)
4. 主题切换工作正常 (浅色 + 深色截图)
5. API 健康检查响应 (curl 输出)
6. 数据库可访问 (迁移状态)
7. 监控仪表板已激活 (仪表板截图)
8. 组件库渲染正常 (组件演示页面)

格式: 附带截图的证据包
判定: PASS / FAIL，附具体问题
```

## 门决策

**双重签署要求**: DevOps 自动化器 (基础设施) + 证据收集器 (视觉)

- **通过**: 可运行骨架，附带完整 DevOps 管道 → 激活阶段 3
- **失败**: 特定基础设施或应用问题 → 修复并重新验证
