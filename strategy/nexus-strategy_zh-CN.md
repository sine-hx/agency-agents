## 1. 战略基础

### 1.1 NEXUS 解决什么

单个智能体很强大。但无协调时，它们产生：
- 冲突的架构决策
- 跨部门重复工作
- 交接边界质量缺口
- 无共享上下文或机构记忆

**NEXUS 消除这些失败模式**，通过定义：
- **谁**在每个阶段激活
- **什么**他们为谁产出
- **何时**他们交接给谁
- **如何**推进前验证质量
- **为什么**每个智能体存在于管道（无乘客）

### 1.2 核心原则

| 原则 | 描述 |
|------|------|
| **管道完整性** | 无阶段通过质量关卡后推进 |
| **上下文连续性** | 每个交接携带完整上下文 — 无智能体冷启动 |
| **并行执行** | 独立工作流并行运行压缩时间线 |
| **证据优于声明** | 所有质量评估需证明，非断言 |
| **快速失败，快速修复** | 每任务最多 3 次重试后升级 |
| **单一真相来源** | 一个规范规格、一个任务列表、一个架构文档 |
