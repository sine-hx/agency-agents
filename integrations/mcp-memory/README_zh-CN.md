# MCP 记忆集成

> 使用模型上下文协议（MCP）为任何智能体提供跨会话的持久记忆。

## 功能介绍

默认情况下，Agency 中的智能体每次会话都从零开始。上下文通过复制粘贴在智能体和会话之间手动传递。MCP 记忆服务器改变了这一点：

- **跨会话记忆**：智能体能记住之前会话的决策、交付物和上下文
- **交接连续性**：当一个智能体交接给另一个时，接收方能准确回忆已完成的工作 — 无需复制粘贴
- **失败时回滚**：当 QA 检查失败或架构决策出错时，回滚到已知良好状态而不是重新开始

## 设置

您需要一个提供记忆工具的 MCP 服务器：`remember`、`recall`、`rollback` 和 `search`。将其添加到您的 MCP 客户端配置（Claude Code、Cursor 等）：

```json
{
  "mcpServers": {
    "memory": {
      "command": "your-mcp-memory-server",
      "args": []
    }
  }
}
```

任何暴露 `remember`、`recall`、`rollback` 和 `search` 工具的 MCP 服务器都可以。请查看 [MCP 生态系统](https://modelcontextprotocol.io) 获取可用实现。

## 如何为任何智能体添加记忆

要为现有智能体增强持久记忆，需在智能体提示中添加**记忆集成**部分。此部分指导智能体在关键时刻使用 MCP 记忆工具。

### 模式

```markdown
## 记忆集成

当您开始会话时：
- 使用您的角色和当前项目作为搜索词，从之前的会话中回忆相关上下文
- 查看任何标记有您智能体名称的记忆，以便从上次中断的地方继续

当您做出关键决策或完成交付物时：
- 用描述性标签（智能体名称、项目、主题）记住决策或交付物
- 包含足够的上下文，以便未来的会话 — 或其他智能体 — 能够理解做了什么以及为什么

当交接给其他智能体时：
- 为接收智能体标记并记住您的交付物
- 包含交接元数据：您完成了什么、还有什么待处理、下一个智能体需要知道什么

当出现失败需要恢复时：
- 搜索最后一个已知良好状态
- 使用回滚恢复到该点，而不是从头重建
```

### 智能体如何使用这些指令

获得这些指令后，LLM 会自动使用 MCP 记忆工具：

- `remember` — 用标签存储决策、交付物或上下文快照
- `recall` — 通过关键词、标签或语义相似性搜索相关记忆
- `rollback` — 出问题时恢复到之前的状态
- `search` — 跨会话和智能体查找特定记忆

无需修改智能体文件的代码。无需编写 API 调用。MCP 工具处理一切。

## 提示

- **一致性标记**：在每条记忆上使用智能体名称和项目名称作为标签。这使回忆可靠。
- **让 LLM 决定什么重要**：记忆指令是指导，不是严格规则。LLM 会判断何时记住和回忆什么。
- **回滚是杀手级功能**：当 Reality Checker 拒绝交付物时，原始智能体可以回滚到上一个检查点，而不是尝试手动撤销更改。
