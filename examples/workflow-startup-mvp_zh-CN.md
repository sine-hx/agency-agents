# 多智能体工作流：初创公司MVP

> 一个逐步示例，展示如何协调多个智能体从想法到交付MVP。

## 场景

你正在构建一个SaaS MVP——一个面向远程团队的团队回顾工具。你有4周时间交付可工作的产品，包含用户注册、核心功能和落地页。

## 智能体团队

| 智能体 | 在此工作流中的角色 |
|-------|---------------------|
| 冲刺优先者 | 将项目拆分为每周冲刺 |
| UX研究员 | 通过快速用户访谈验证想法 |
| 后端架构师 | 设计API和数据模型 |
| 前端开发者 | 构建React应用 |
| 快速原型者 | 快速让第一版本运行起来 |
| 增长黑客 | 在构建时规划发布策略 |
| 现实检验者 | 在进入下阶段前把关每里程碑 |

## 工作流

### 第1周：发现 + 架构

**步骤1 —— 激活冲刺优先者**

```
激活冲刺优先者。

时间线：4周到MVP发布。
核心功能：用户认证、创建回顾板、添加卡片、投票、行动项。
约束：独立开发者、React + Node.js技术栈、部署到Vercel + Railway。

将此拆分为4个每周冲刺，带清晰交付物和验收标准。
```

**步骤2 —— 激活UX研究员（并行）**

```

竞品：EasyRetro、Retrium、Parabol。

运行快速竞争分析并识别：
1. 哪些功能是基准
2. 竞品何处不足
3. 我们可以独占的一个差异化点

输出1页研究简报。
```

**步骤3 —— 交接给后端架构师**

```
激活后端架构师。

这是我们的冲刺计划：[粘贴冲刺优先者输出]
这是我们的研究简报：[粘贴UX研究员输出]

交付：
1. 数据库架构（SQL）
2. REST API端点列表
3. 用于实时板更新的WebSocket事件

```

### 第3周：打磨 + 落地页

**步骤6 —— 前端开发者继续，增长黑客开始**

```

预算：0美元（仅有机发布）。

创建发布计划：
1. 落地页文案（hero、特性、CTA）
2. 发布渠道（Product Hunt、Reddit、Hacker News、Twitter）
3. 逐日发布序列
4. 第1周要追踪的指标
```

### 第4周：发布

**步骤7 —— 最终现实检验**

```
激活现实检验者。

- 在线URL：[url]

- 错误监控：Sentry已配置
- 数据库备份：每日自动

过一遍发布检查清单并给出GO / NO-GO决策。
每标准需证据。
```

## 关键模式

1. **顺序交接**：每智能体输出成为下一智能体输入
2. **并行工作**：UX研究员和冲刺优先者可在第1周同时运行
3. **质量门**：中点和发布前的现实检验者防止发布损坏代码
4. **上下文传递**：总是在下一提示中粘贴前智能体输出——智能体不共享记忆

## 提示

- 在步骤间复制粘贴智能体输出——不要总结，使用完整输出
- 如果现实检验者标记问题，循环回相关专家修复
- 熟悉手动版本后，考虑使用Orchestrator智能体自动化此流程
