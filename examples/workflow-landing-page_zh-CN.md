# 多智能体工作流：落地页冲刺

> 使用4个智能体在一天内交付转化优化的落地页。

## 场景

你需要为新产品发布准备一个落地页。它需要外观出色、转化访客，并在当天结束前上线。

## 智能体团队

| 智能体 | 在此工作流中的角色 |
|-------|---------------------|
| 内容创作者 | 撰写文案 |
| UI设计师 | 设计布局和组件规格 |
| 前端开发者 | 构建页面 |
| 增长黑客 | 优化转化 |

## 工作流

### 上午：文案 + 设计（并行）

**步骤1a —— 激活内容创作者**

```
激活内容创作者。

为"FlowSync"撰写落地页文案——一个能在5分钟内连接任意两个SaaS工具的API集成平台。

目标受众：中型公司的开发者和技术PM。
语调：自信、简洁、略带俏皮。

需要的部分：
1. Hero（标题 + 副标题 + CTA）
2. 问题陈述（3个痛点）
3. 工作原理（3步）
4. 社会证明（占位testimonial格式）
5. 定价（3档：免费、Pro、企业）
6. 最终CTA

保持可扫描。无废话。
```

**步骤1b —— 激活UI设计师（并行）**

```
激活UI设计师。

为SaaS落地页设计规格。产品：FlowSync（API集成平台）。
风格：干净、现代、暗色模式选项。参考Linear或Vercel美学。

交付：
1. 布局线框（部分顺序 + 间距）
2. 色彩调板（主色、次色、强调色、背景）
3. 字体排印（字体配对、标题尺寸、正文尺寸）
4. 组件规格：hero部分、特性卡片、定价表、CTA按钮
5. 响应式断点（移动、平板、桌面）
```

### 中午：构建

**步骤2 —— 激活前端开发者**

```
激活前端开发者。

根据这些规格构建落地页：

文案：[粘贴内容创作者输出]
设计：[粘贴UI设计师输出]

技术栈：HTML、Tailwind CSS、最小原生JS（无需框架）。
要求：
- 响应式（移动优先）
- 快速（无重型资源，系统字体可接受）
- 无障碍（适当标题、alt文本、焦点状态）
- 包含工作的邮件注册表单（动作URL：/api/subscribe）

交付单个index.html文件，可直接部署。
```

### 下午：优化

**步骤3 —— 激活增长黑客**

```
激活增长黑客。

审阅此落地页以进行转化优化：

[粘贴HTML或描述当前页面]

评估：
1. CTA是否在首屏上方？
2. 价值主张是否在5秒内清晰？
3. 注册流程中是否有摩擦？
4. 你首先会运行哪些A/B测试？
5. SEO基础：meta标签、OG标签、结构化数据

给我具体改动，而非泛泛建议。
```

## 时间线

| 时间 | 活动 | 智能体 |
|------|----------|-------|
| 9:00 | 文案 + 设计启动（并行） | 内容创作者 + UI设计师 |
| 11:00 | 构建开始 | 前端开发者 |
| 14:00 | 第一版本就绪 | — |
| 14:30 | 转化审阅 | 增长黑客 |
| 15:30 | 应用反馈 | 前端开发者 |
| 16:30 | 发布 | 部署到Vercel/Netlify |

## 关键模式

1. **并行启动**：文案和设计同时进行，因为它们彼此独立
2. **合并点**：前端开发者需等待两者输出后再开始
3. **反馈循环**：增长黑客审阅，然后前端开发者应用改动
4. **时间框定**：每步有清晰时间框，防止范围蔓延
