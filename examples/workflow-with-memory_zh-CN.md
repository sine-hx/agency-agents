# 多智能体工作流：带持久记忆的初创公司MVP

> 与[workflow-startup-mvp.md](workflow-startup-mvp.md)相同的初创公司MVP工作流，但使用MCP记忆服务器处理智能体间状态。无需复制粘贴交接。

## 手动交接的问题

在标准工作流中，每个智能体到智能体的过渡如下：

```
激活后端架构师。

这是我们的冲刺计划：[粘贴冲刺优先者输出]
这是我们的研究简报：[粘贴UX研究员输出]

为RetroBoard设计API和数据库架构...
```

你是粘合剂。你在智能体间复制粘贴输出，跟踪已完成事项，并希望不丢失上下文。这在小项目中可行，但在以下情况崩溃：

- 会话超时，你丢失输出
- 多个智能体需要相同上下文
- QA失败，你需要回滚到前状态
- 项目跨越数天或数周，跨越多次会话

## 解决方案

安装MCP记忆服务器后，智能体将交付物存储在记忆中并自动检索所需内容。交接变为：

```
激活后端架构师。

项目：RetroBoard。召回此项目的先前上下文
并设计API和数据库架构。
```

智能体搜索记忆中的RetroBoard上下文，找到前智能体存储的冲刺计划和研究简报，并从那里继续。

## 设置

安装任何支持`remember`、`recall`和`rollback`操作的MCP兼容记忆服务器。参见[integrations/mcp-memory/README.md](../integrations/mcp-memory/README.md)进行设置。

## 场景

与标准工作流相同：一个SaaS团队回顾工具（RetroBoard），4周到MVP，独立开发者。

## 智能体团队

| 智能体 | 在此工作流中的角色 |
|-------|---------------------|
| 冲刺优先者 | 将项目拆分为每周冲刺 |
| UX研究员 | 通过快速用户访谈验证想法 |
| 后端架构师 | 设计API和数据模型 |
| 前端开发者 | 构建React应用 |
| 快速原型者 | 快速让第一版本运行起来 |
| 增长黑客 | 在构建时规划发布策略 |
| 现实检验者 | 在进入下阶段前把关每里程碑 |

每智能体的提示中有一个记忆集成部分（参见[integrations/mcp-memory/README.md](../integrations/mcp-memory/README.md)了解如何添加）。

## 工作流

### 第1周：发现 + 架构

**步骤1 —— 激活冲刺优先者**

```
激活冲刺优先者。

项目：RetroBoard——面向远程团队的实时团队回顾工具。
时间线：4周到MVP发布。
核心功能：用户认证、创建回顾板、添加卡片、投票、行动项。
约束：独立开发者、React + Node.js技术栈、部署到Vercel + Railway。

将此拆分为4个每周冲刺，带清晰交付物和验收标准。
完成后记住你的冲刺计划，标记此项目。
```

冲刺优先者产出冲刺计划并存储在记忆中，标记`sprint-prioritizer`、`retroboard`和`sprint-plan`。

**步骤2 —— 激活UX研究员（并行）**

```
激活UX研究员。

我正在构建面向远程团队（5-20人）的团队回顾工具。
竞品：EasyRetro、Retrium、Parabol。

运行快速竞争分析并识别：
1. 哪些功能是基准
2. 竞品何处不足
3. 我们可以独占的一个差异化点

输出1页研究简报。完成后记住它，标记此项目。
```

UX研究员存储研究简报，标记`ux-researcher`、`retroboard`和`research-brief`。

**步骤3 —— 交接给后端架构师**

```
激活后端架构师。

项目：RetroBoard。召回前智能体的冲刺计划和研究简报。
技术栈：Node.js、Express、PostgreSQL、Socket.io用于实时。

设计：
1. 数据库架构（SQL）
2. REST API端点列表
3. 用于实时板更新的WebSocket事件
4. 认证策略推荐

记住每交付物，标记此项目和frontend-developer。
```

后端架构师自动从记忆召回冲刺计划和研究简报。无需复制粘贴。它存储架构和API规格，标记`backend-architect`、`retroboard`、`api-spec`和`frontend-developer`。

### 第2周：构建核心功能

**步骤4 —— 激活前端开发者 + 快速原型者**

```
激活前端开发者。

项目：RetroBoard。召回后端架构师的API规格和架构。

构建RetroBoard React应用：
- 技术栈：React、TypeScript、Tailwind、Socket.io-client
- 页面：登录、仪表板、板视图
- 组件：RetroCard、VoteButton、ActionItem、BoardColumn

从板视图开始——这是核心体验。
聚焦实时：当一位用户添加卡片，所有人看到。
记住你的进度，标记此项目。
```

前端开发者从记忆拉取API规格并据此构建。

**步骤5 —— 中点现实检验**

```
激活现实检验者。

项目：RetroBoard。我们在4周MVP构建的第2周。

召回此项目前智能体的所有交付物。

评估：
1. 我们能否现实地在再2周内发布？
2. 我们应该削减什么以赶上截止？
3. 有什么技术债务会在发布时困扰我们？

记住你的裁决，标记此项目。
```

现实检验者完整可见迄今产生的所有内容——冲刺计划、研究简报、架构、API规格和前端进度——无需你收集并粘贴全部。

### 第3周：打磨 + 落地页

**步骤6 —— 前端开发者继续，增长黑客开始**

```
激活增长黑客。

产品：RetroBoard——团队回顾工具，1周内发布。
目标：远程优先公司的工程经理和敏捷教练。
预算：0美元（仅有机发布）。

召回项目上下文和现实检验者的裁决。

创建发布计划：
1. 落地页文案（hero、特性、CTA）
2. 发布渠道（Product Hunt、Reddit、Hacker News、Twitter）
3. 逐日发布序列
4. 第1周要追踪的指标

记住发布计划，标记此项目。
```

### 第4周：发布

**步骤7 —— 最终现实检验**

```
激活现实检验者。

项目：RetroBoard，准备发布。

召回所有项目上下文、前裁决和发布计划。

评估生产就绪度：
- 在线URL：[url]
- 测试账户已创建：是
- 错误监控：Sentry已配置
- 数据库备份：每日自动

过一遍发布检查清单并给出GO / NO-GO决策。
每标准需证据。
```

### 当QA失败时：回滚

在标准工作流中，当现实检验者拒绝交付物，你回到责任智能体并尝试解释何处出错。带记忆后，恢复循环更紧凑：

```
激活后端架构师。

项目：RetroBoard。现实检验者标记了API设计问题。
召回现实检验者的反馈和你前API规格。
回滚到你最后已知良好架构并解决提出的具体问题。
完成后记住更新的交付物。
```

后端架构师可确切看到现实检验者标记了什么，召回自己的前工作，回滚到检查点，并产出修复——无需你手动追踪版本。

## 前后对比

| 方面 | 标准工作流 | 带记忆 |
|--------|------------------|-------------|
| **交接** | 在智能体间复制粘贴完整输出 | 智能体自动召回所需 |
| **上下文丢失** | 会话超时丢失一切 | 记忆跨会话持久 |
| **多智能体上下文** | 手动从N智能体汇编上下文 | 智能体搜索项目标签 |
| **QA失败恢复** | 手动描述何处出错 | 智能体召回反馈 + 回滚 |
| **多日项目** | 每会话重新建立上下文 | 智能体从中断处继续 |
| **所需设置** | 无 | 安装MCP记忆服务器 |

## 关键模式

1. **用项目名标记一切**：这是使召回工作的关键。每记忆获标记`retroboard`（或你的项目名）。
2. **为接收智能体标记交付物**：当后端架构师完成API规格，它用`frontend-developer`标记记忆，使前端开发者召回时找到。
3. **现实检验者获完整可见**：因所有智能体将工作存储在记忆，现实检验者可召回项目全部内容，无需你汇编。
4. **回滚替代手动撤销**：当某事失败，回滚到最后检查点，而非试图弄清什么变了。

## 提示

- 你无需一次修改每智能体。从最常用智能体开始添加记忆集成，然后扩展。
- 记忆指令是提示，非代码。LLM解读它们并按需调用MCP工具。你可调整措辞匹配你的风格。
- 任何支持`remember`、`recall`、`rollback`和`search`工具的MCP兼容记忆服务器将与此工作流工作。
