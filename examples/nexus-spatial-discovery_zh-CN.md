## 1. 机会发现

### 如何发现机会

跨多个来源的网络研究发现了三个趋同趋势：

- **AI基础设施/编排** 是增长最快的软件类别（AI编排市场2026年估值约135亿美元，复合年增长率22%以上）
- **空间计算**（Vision Pro、WebXR）正在成熟，但缺乏杀手级企业应用
- 每个现有的AI工作流工具（LangSmith、n8n、Flowise、CrewAI）都是**扁平的2D仪表板**

### 产品概念：Nexus Spatial

一个空间计算中的AI智能体指挥中心——一个VisionOS + WebXR应用程序，提供沉浸式3D指挥中心，用于编排、监控和与AI智能体交互。用户将智能体管道可视化为3D节点图，在空间面板中监控实时输出，在3D空间中通过拖放构建工作流，并在共享空间环境中协作。

### 为什么这个代理机构具有独特优势

该代理机构拥有深厚的空间计算专业知识（XR开发者、VisionOS工程师、Metal专家、界面架构师），同时具备完整的工程、设计、营销和运营团队——对于既需要空间计算 mastery 又需要企业软件 rigor 的产品来说，这是一种罕见的组合。

## 3. 技术架构

**智能体：** 后端架构师

### 系统概览

8服务架构，具有清晰的权责边界，设计用于水平扩展和供应商无关的AI集成。

```
+------------------------------------------------------------------+
|                     客户端层                                       |
|  VisionOS原生 (Swift/RealityKit)  |  WebXR (React Three Fiber) |
+------------------------------------------------------------------+
                              |
+-----------------------------v------------------------------------+
|                      API网关 (Kong / AWS API GW)                  |
|  速率限制 | JWT验证 | WebSocket升级 | TLS                       |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                      服务层                                       |
|  Auth | Workspace | Workflow | Orchestration (Rust) |             |
|  Collaboration (Yjs CRDT) | Streaming (WS) | Plugin | Billing    |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                      数据层                                       |
|  PostgreSQL 16 | Redis 7集群 | S3 | ClickHouse | NATS           |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                    AI供应商层                                     |
|  OpenAI | Anthropic | Google | 本地模型 | 自定义插件               |
+------------------------------------------------------------------+
```

### 技术栈

| 组成部分 | 技术 | 理据 |
|-----------|------------|-----------|
| 编排引擎 | **Rust** | 亚毫秒调度，零GC暂停，智能体沙箱化的内存安全 |
| API服务 | TypeScript / NestJS | CRUD密集服务的开发者效率 |
| VisionOS客户端 | Swift 6, SwiftUI, RealityKit | 一流的空间计算，支持Liquid Glass |
| WebXR客户端 | TypeScript, React Three Fiber | 生产级WebXR，带React组件模型 |
| 消息代理 | NATS JetStream | 轻量，精确一次交付，比Kafka简单 |
| 协作 | Yjs (CRDT) + WebRTC | 无冲突并发3D图形编辑 |
| 主数据库 | PostgreSQL 16 | JSONB用于灵活配置，行级安全用于租户隔离 |

### 核心数据模型

14张表涵盖：
- **身份与访问：** users, workspaces, team_memberships, api_keys
- **工作流：** workflows, workflow_versions, nodes, edges
- **执行：** executions, execution_steps, step_output_chunks
- **协作：** collaboration_sessions, session_participants
- **凭证：** provider_credentials（AES-256-GCM加密）
- **计费：** subscriptions, usage_records
- **审计：** audit_log（仅追加）

### 节点类型注册表

```
内置节点类型：
  ai_agent          -- 使用提示调用AI供应商
  prompt_template   -- 使用变量渲染模板
  conditional       -- 基于表达式路由
  transform         -- 沙箱化代码片段（JS/Python）
  input / output    -- 工作流入口/出口点
  human_review      -- 暂停等待人工审批
  loop              -- 重复子图
  parallel_split    -- 分发到分支
  parallel_join     -- 等待分支完成
  webhook_trigger   -- 外部HTTP触发器
  delay             -- 定时暂停
```

### WebSocket通道

通过WSS进行实时流式传输，具备：
- 每通道序列号用于排序
- 间隙检测与重放请求
- 当落后超过1000事件时快照恢复
- 客户端侧节流用于低功耗设备

### 安全架构

| 层级 | 机制 |
|-------|-----------|
| 用户认证 | OAuth 2.0（GitHub、Google、Apple）+ 电子邮件/密码 + 可选TOTP MFA |
| API密钥 | SHA-256哈希，范围限制，可选过期 |
| 服务间通信 | 通过服务网格的mTLS |
| WebSocket认证 | 一次性票据，30秒过期 |
| 凭证存储 | 信封加密（AES-256-GCM + AWS KMS） |
| 代码沙箱 | gVisor/Firecracker微VM（无网络，256MB RAM，30秒CPU） |
| 租户隔离 | PostgreSQL行级安全 + S3 IAM策略 + NATS主题范围限制 |

## 4. 品牌策略

**智能体：** 品牌守护者

### 定位

**品类创造而非品类竞争。** Nexus Spatial定义了一个新品类——**空间AI运营（SpatialAIOps）**——而不是在拥挤的AI可观测性仪表板空间中争夺位置。

**定位声明：** 对于管理复杂AI智能体工作流的技术团队，Nexus Spatial是沉浸式3D指挥中心，提供智能体编排的空间感知，不同于扁平2D仪表板，因为空间计算将监控从阅读仪表板转变为栖息于你的基础设施之中。

### 名称验证

"Nexus Spatial"已**验证为强劲：**
- "Nexus"连接到NEXUS编排框架（专家网络，策略统一）
- "Nexus"独立含义为"中央连接点"——完美契合指挥中心
- "Spatial"是Apple和行业已标准化的行业标准描述词
- 音素平衡：三个音节，然后两个
- **需行动：** 在尼斯分类第9、42和38类进行商标清查

### 关键发现：调试是杀手级用例

运行时追踪在工作流结构上的空间叠加解决了一个真正、量化的痛点，没有2D工具能良好处理。此工作流应获得最多设计和工程投入。

### 关键设计洞察

空间为**结构性**任务（放置、连接、重排节点）增加价值，但为**参数性**任务（文本输入、配置）制造摩擦。界面必须无缝融合空间和2D模式——锚定到空间位置的2D面板。

### 7项设计原则

1. **空间需证明其价值** —— 如果2D更清晰，使用2D。每次审阅应问："这扁平会更好吗？"
2. **先可扫视再可检查** —— 关键信息在2秒内通过颜色、大小、运动、位置可感知
3. **免手是基线** —— 凝视 + 语音覆盖所有阅读/导航操作；手增加精度但非必需
4. **尊重认知重力** —— 扩展2D心智模型（从左到右流），而非替代；z轴添加层叠
5. **渐进式空间复杂度** —— 新用户从近乎2D开始；空间能力随信心增长揭示
6. **物理隐喻，数字能力** —— 节点被"拾起"（物理）但也可复制和版本化（数字）
7. **静默是特性** —— 健康系统感觉平静；颜色和运动信号偏离正常

## 9. 空间界面架构

**智能体：** XR界面架构师

### 指挥剧院

工作区组织为围绕用户的弯曲剧院：

```
                        概览顶篷
                     （管道拓扑）
                    ~~~~~~~~~~~~~~~~~~~~~~~~
                   /                        \
                  /     聚焦弧（120度）       \
                 /    主要节点图工作          \
                /________________________________\
               |                                  |
    左侧       |        用户位置                  |       右侧
    工具       |        （原点0,0,0）            |       工具
    边栏       |                                  |       边栏
               |__________________________________|
                \                                /
                 \      架子（视线以下）          /
                  \   智能体状态，快捷工具       /
                   \_________________________ /
```

- **聚焦弧**（120度，1.2-2.0米）：主要节点图工作区
- **概览顶篷**（上方，2.5-4.0米）：迷你管道拓扑 + 健康热图
- **工具边栏**（左/右侧）：智能体库、监控、日志
- **架子**（视线以下，0.8-1.0米）：运行/停止、撤销/重做、快捷工具

### 三层深度系统

| 层级 | 深度 | 内容 | 不透明度 |
|-------|-------|---------|---------|
| 前景 | 0.8 - 1.2米 | 活动面板、检查器、模态 | 100% |
| 中景 | 1.2 - 2.5米 | 节点图、连接、工作区 | 100% |
| 背景 | 2.5 - 5.0米 | 概览地图、环境状态 | 40-70% |

### 3D中的节点图

**数据流向用户流动。** 节点按执行顺序沿z轴排列：

```
用户（此处）
  z=0.0米   [输出节点]     -- 结果
  z=0.3米   [变换节点]     -- 处理器
  z=0.6米   [智能体节点]   -- LLM调用
  z=0.9米   [检索节点]     -- RAG、API
  z=1.2米   [输入节点]     -- 触发器
```

并行分支水平展开（x轴）。条件分支垂直展开（y轴）。

**节点表示（3 LOD）：**
- **LOD-0**（休息，>1.5米）：12x8厘米磨砂玻璃矩形，带类型图标、名称、状态发光
- **LOD-1**（悬停，400毫秒凝视）：扩展到14x10厘米，揭示端口、上次运行信息
- **LOD-2**（选中）：滑到前景，扩展到30x40厘米详情面板，带实时配置编辑

**连接作为发光管道：**
- 休息时4毫米直径，承载数据时8毫米
- 按数据类型颜色编码（白色=文本，青色=结构化，紫红色=图像，琥珀色=音频，绿色=工具调用）
- 动画粒子显示流向和速度
- 当>3条在同层间并行时自动捆绑

### 7种智能体状态

| 状态 | 边缘发光 | 内部 | 声音 | 粒子 |
|-------|-----------|----------|-------|-----------|
| Idle | 稳定绿色，低 | 静态磨砂玻璃 | 无 | 无 |
| Queued | 脉动琥珀，1Hz | 微弱旋转 | 无 | 输入处缓慢漂移 |
| Running | 稳定蓝色，中 | 动画闪烁 | 柔和空间嗡声 | 连接上快速流动 |
| Streaming | 蓝 + 输出流 | 闪烁 + 文本片段 | 嗡声 | 文本片段向前流动 |
| Completed | 白色闪烁，然后绿色 | 静态 | 完成提示音 | 无 |
| Error | 脉动红色，2Hz | 红色调 | 警报音（一次） | 无 |
| Paused | 稳定琥珀 | 冻结帧 + 暂停图标 | 无 | 原位冻结 |

| 窗口到沉浸 | 1000毫秒 | 边框溶解，节点扩展到完整空间位置 |

### 舒适措施

- 无用户操作时相机不主动移动
- 稳定地平线（水平面永不倾斜）
- 主要交互在0.8-2.5米，视线+/-15度内
- 45分钟后休息提示（环境照明变化，非模态）
- 快速移动时周边晕影
- 所有常用控制可手臂垂侧时访问（仅手腕/手指）

## 10. 跨智能体综合

### 所有8智能体的共识点

1. **2D优先，空间其次。** 每个智能体独立得出此结论。先构建优秀的Web仪表板，然后渐进添加空间能力。

2. **调试是杀手级用例。** 产品研究员、UX研究员和XR界面架构师都汇聚于此：运行时追踪在工作流结构上的空间叠加是3D真正胜过2D之处。

3. **WebXR优于VisionOS用于初期覆盖。** Vision Pro的约100万安装基数无法支撑业务。浏览器中的WebXR是分发解锁。

4. **"作战室"协作场景。** 多个智能体强调协作事件响应为最强空间价值主张——团队进入共享3D空间一起调试失败管道。

5. **渐进披露至关重要。** UX研究、空间UI和支持都强调空间复杂度必须渐进揭示，绝不一次性倾倒给首次用户。

6. **语音作为强力用户加速器。** UX研究员和XR界面架构师都识别语音命令为"空间计算的命令行"——对无障碍和专家效率至关重要。
